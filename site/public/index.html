What is plshelp?

plshelp is an open source CLI that turns documentation websites, Markdown, and plain text files into a local searchable corpus for coding assistants and RAG workflows. It handles crawling, cleaning, chunking, and indexing — so your agent has accurate context without hitting the internet at query time.

Why plshelp?
Local-first Indexed docs live in SQLite and local files on your machine. No cloud sync, no vendor lock-in.
Built for documentation Crawls docs sites, strips boilerplate, preserves technical content, exports clean .md and .txt.
Hybrid retrieval Use BM25, embeddings, or hybrid ranking. Tune the blend based on your workflow and latency budget.
Agent-friendly Query from the terminal or pipe --json output directly into Codex, Claude Code, or your own tooling.
Bring your own sources Index docs sites, local Markdown, or plain text. Merge multiple libraries into one corpus.
Transparent storage Your DB, artifacts, model cache, and config are all visible, portable, and configurable.
What can I do with plshelp?
  • Crawl a docs site and query it locally
  • Build local context for coding agents
  • Export cleaned docs as docs.md and docs.txt
  • Merge multiple libraries into one searchable corpus
  • Index internal notes or project docs from local files
Example workflows
Index a docs site
plshelp add nextjs https://nextjs.org/docs
plshelp query nextjs "how does app router work"
Index a local Markdown file
plshelp index mynotes --file ./notes/rust.md
plshelp query mynotes "what did i write about ownership"
Merge multiple libraries
plshelp merge numpy numpyuser numpyref
plshelp query numpy "what is broadcasting"
Export cleaned docs
plshelp export numpyuser
# → artifacts/numpyuser/docs.md
# → artifacts/numpyuser/docs.txt
FAQ
What can I index?
  • documentation websites
  • local Markdown files
  • local text files
  • merged collections of multiple libraries
Does plshelp require a hosted vector database?
No. It uses local SQLite storage with local embedding generation. Everything runs on your machine.
Can I use it without embeddings?
Yes. BM25 search works immediately after chunking. Embeddings are optional for hybrid or vector-only retrieval.
Does it work well for coding agents?
Yes. Designed for local documentation context with structured --json output for agent tooling.
Where does data live?
By default in your local application data directory:
  • SQLite database
  • doc artifacts (.md / .txt)
  • embedding model cache
  • config file
Is plshelp open source?
Yes. MIT licensed and available on GitHub.
