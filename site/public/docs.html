Minimal docs for everything that works today

plshelp crawls documentation, indexes local files, exports cleaned artifacts, and returns local context through BM25, embeddings, or hybrid retrieval.

curl -fsSL https://plshelp.run/install.sh | sh
npm install -g plshelp
bun add -g plshelp
brew install plshelp
What it does

plshelp is a local-first documentation engine. It crawls docs sites, compiles cleaned markdown and text artifacts, chunks and embeds content locally, and answers queries against that local corpus. It also supports local Markdown and text files, merged libraries, and structured --json output for agent tooling.

Quickstart
Index a docs site
plshelp add nextjs https://nextjs.org/docs
plshelp query nextjs "how does app router work"
Index a local Markdown file
plshelp index mynotes --file ./notes/rust.md
plshelp query mynotes "what did i write about ownership"
Merge multiple libraries
plshelp merge numpy numpyuser numpyref
plshelp query numpy "what is broadcasting"
Export cleaned docs
plshelp export numpyuser
# → artifacts/numpyuser/docs.md
# → artifacts/numpyuser/docs.txt
Command surface
addCrawl a documentation site, chunk it, embed it, and make it queryable in one command.
crawlOnly crawl and compile pages for a library without chunking or embedding yet.
indexChunk and embed an existing library, or index a local file via --file.
chunkGenerate parent and child chunks and build the lexical index.
embedGenerate embeddings for child chunks using the configured local model.
querySearch one library using hybrid, vector-only, or keyword-only retrieval.
traceInspect ranking internals, scores, and chunk/parent provenance for a query.
askRun one question across multiple libraries or the entire local corpus.
mergeCreate a merged logical library from multiple indexed libraries.
exportWrite the compiled documentation artifacts back out as docs.md and docs.txt.
show / listInspect library stats, rollups, and what is currently indexed locally.
openOpen a child chunk and its parent metadata by chunk id.
Retrieval model
KeywordBM25-style lexical retrieval works immediately after chunking and does not require embeddings.
VectorEmbeddings run on child chunks and return the matching parent text for coherence.
HybridThe default mode blends lexical and vector scores, with weights controlled by config.
JSON outputUse --json on read and write commands for machine-readable output in agents and scripts.
Configuration

plshelp creates and reads a local config.toml at startup. You can configure paths, embedding model and batch size, parent and child chunk sizes, retrieval defaults, SQLite settings, and ONNX threading without touching the code.

By default, runtime data lives in your local application data directory. On macOS that is typically ~/Library/Application Support/plshelp, on Linux it is usually ~/.local/share/plshelp or the XDG data directory, and on Windows it would normally live under %APPDATA%\plshelp.
Registry

The plshelp registry is a static collection of pre-crawled documentation artifacts. Each entry exposes compiled docs.md and docs.txt files plus a public index.json manifest that the website can browse without loading raw docs until a user requests them.
