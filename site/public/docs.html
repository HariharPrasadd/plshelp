Introduction

Get started with plshelp.

Overview

plshelp is an open source CLI tool that turns documentation websites, Markdown files, and plain text into a local searchable corpus for coding assistants and RAG workflows. It handles crawling, cleaning, chunking, and indexing — so your agent has accurate context without hitting the internet at query time.

Everything runs on your machine. No hosted database, no API calls at retrieval time, no vendor lock-in. Your indexed docs live in a local SQLite database alongside human-readable .md and .txt artifacts.

Note

plshelp is in active development. Commands and config options may change between minor versions. Check the changelog before upgrading.

Installation

Install plshelp using any of the following methods.

curl (macOS / Linux)

shell 
# Download and run the install script
curl -fsSL https://plshelp.run/install.sh | sh

npm / bun

shell 
npm install -g plshelp
# or
bun add -g plshelp

Homebrew

shell 
brew install plshelp

Windows (PowerShell)

powershell 
irm https://plshelp.run/install.ps1 | iex

Quickstart

Index a documentation site and query it in two commands.

shell 
# Crawl and index the Next.js docs
plshelp add nextjs https://nextjs.org/docs

# Query with natural language
plshelp query nextjs "how does app router work"

# Output as JSON for agent consumption
plshelp query nextjs "how does app router work" --json

The add command crawls the site, strips boilerplate, chunks the content, generates embeddings, and writes everything to your local SQLite database. Subsequent queries hit only local storage — no network required.

Query flags

--json Output results as newline-delimited JSON. Suitable for piping into agent tooling.
--top N Return the top N results (default: 5).
--mode Retrieval mode: bm25, vector, or hybrid (default).
--alpha Blend weight for hybrid mode. 0.0 = pure BM25, 1.0 = pure vector (default: 0.5).
